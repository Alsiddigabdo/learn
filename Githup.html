Git نظام تحكم في الإصدارات

الـ Git هو نظام تحكم في الإصدارات يستخدم لتتبع التغييرات التي تطرأ على ملفات المشروع عبر الوقت. يعتبر Git أداة قوية تتيح للمطورين التعاون في العمل وإدارة الإصدارات بشكل فعال.

GitHub part 1

  1. تثبيت Git: قم بتثبيت Git على جهاز الكمبيوتر الخاص بك من خلال تنزيله من الموقع الرسمي لـ Git (https://git-scm.com/downloads). بعد التثبيت، يمكنك فتح تطبيق السطر الأوامر أو Git Bash لإجراء العمليات المطلوبة.
  2. إنشاء مستودع (Repository): قم بإنشاء مستودع Git جديد لمشروعك. يمكنك القيام بذلك عن طريق فتح مجلد جديد على جهازك وتشغيل الأمر git init لتهيئة المجلد كمستودع Git.
  3. إضافة الملفات: بعد إنشاء المستودع، قم بإضافة الملفات التي ترغب في تتبعها عن طريق استخدام الأمر git add. يمكنك إضافة ملف واحد بواحد أو جميع الملفات الموجودة في المجلد باستخدام git add ..
  4. إجراء تعديلات: بمجرد إضافة الملفات، يمكنك تعديلها بحرية. عندما تنتهي من تعديلاتك، يمكنك استخدام الأمر git status لمراجعة التغييرات التي تم إجراؤها.
  5. إنشاء نسخة احتياطية (Commit): بعد الانتهاء من التعديلات والتأكد من أن كل التغييرات الضرورية تمت إضافتها، قم بإجراء عملية الـ Commit لحفظ هذه التغييرات. استخدم الأمر git commit -m "رسالة الـCommit" لإنشاء نسخة احتياطية ووصف التغييرات التي تمت.
  6. التعاون والتحديثات: يمكنك تحميل مشروعك إلى مستودع Git على خدمة استضافة مثل GitHub أو GitLab للتعاون مع الفريق. يمكن لأعضاء الفريق تنزيل المشروع وإجراء تعديلاتهم وإرسال التحديثات التي يمكنك دمجها في مشروعك بسهولة. 

# Examlpe code for initializing a git repository
git init

# Add files to staging area
git add .

# Commit the changes
git commit -m "Initial commit"

GitHub part2

  1. استعراض التاريخ (Log): يمكنك استخدام أمر git log لعرض سجل التغييرات في مشروعك. ستشاهد تفاصيل الـ Commits التي تمت، بما في ذلك رسالة الـ Commit والمؤلف والتاريخ.
  2. الانتقال بين الفروع (Branches): الفروع هي نسخ من المشروع الأساسي تسمح لك بعمل تجارب مستقلة على التغييرات. يمكنك إنشاء فرع جديد باستخدام الأمر git branch <اسم الفرع> والتنقل بين الفروع باستخدام الأمر git checkout <اسم الفرع>.
  3. الدمج (Merge): عندما تكون راضيًا عن التغييرات على فرع معين وترغب في دمجها مع الفرع الرئيسي، يمكنك استخدام الأمر git merge <اسم الفرع> لدمج التغييرات.
  4. حل التعارضات (Resolving Conflicts): في بعض الأحيان قد يحدث تعارض بين التغييرات المختلفة في الفروع المختلفة. لحل هذه التعارضات، يمكنك استخدام أدوات دمج مثل محرر النصوص المفضل لديك لتعديل الملفات وحل التعارضات يدويًا.
  5. الانتقال إلى تعيين سابق (Checkout): يمكنك استخدام الأمر git checkout <commit_id> للانتقال إلى حالة معينة في التاريخ وعرض حالة المشروع في تلك الفترة.
  6. رفع التغييرات (Push): بعد الانتهاء من إجراء التغييرات اللازمة في مشروعك على الفرع الحالي، يمكنك رفع (Push) هذه التغييرات إلى مستودع Git على خدمة الاستضافة (مثل GitHub أو GitLab) باستخدام الأمر git push.
  7. سحب التحديثات (Pull): إذا كان لديك فروع متعددة من مشروعك وترغب في تحديث فرعك المحلي بالتحديثات الأخيرة من الفرع الرئيسي، يمكنك استخدام الأمر git pull لجلب التحديثات الجديدة ودمجها مع فرعك المحلي. 

# Example code for git log, git branch, git merge, git checkout, git push, and git pull
git log
git branch new_feature_branch
git checkout new_feature_branch
git merge main_branch
git checkout main_branch
git push origin main_branch
git pull origin main_branch

GitHub رفع مشروع على

  1. طريقة رفع مشروعك على GitHub:
    1. إنشاء حساب GitHub: قم بإنشاء حساب على GitHub إذا كنت لا تملك حسابًا بالفعل، قم بتسجيل الدخول إليه.
    2. إنشاء مستودع (Repository): قم بإنشاء مستودع جديد على GitHub. يمكنك تسميته على أنه اسم مشروعك واختيار الإعدادات المناسبة.
    3. توصيل المشروع بالمستودع: افتح موجه الأوامر (Command line) أو Git Bash وانتقل إلى دليل مشروعك المحلي. ثم قم بتنفيذ الأوامر التالية لتهيئة المشروع للعمل مع GitHub: 
      
# Initialize the local repository
git init

# Add files to the staging area
git add .

# Commit the changes
git commit -m "Initial commit"
    4. ربط المشروع المحلي بالمستودع البعيد: استخدم الأمر التالي لربط المشروع المحلي بمستودع GitHub البعيد: 
      
git remote add origin <repository-url>
      حيث <repository-url> هو رابط URL لمستودع GitHub الذي قمت بإنشائه.
    5. رفع المشروع إلى GitHub: استخدم الأمر التالي لرفع مشروعك إلى GitHub: 
      
git push -u origin master
      سيتم رفع المشروع إلى GitHub ويرتبط مع المستودع الخاص بك.
